# Security policy

## Supported versions

Vernier has not published a supported release yet. Security fixes apply to the
latest state of the default branch until a version support policy is announced.

## Reporting a vulnerability

Use GitHub private vulnerability reporting for this repository. Do not open a
public issue and do not include secrets, private endpoints, wallet material, or
operational data in public discussions.

Include the affected revision, impact, reproduction steps, and any suggested
mitigation. Please allow the maintainer time to confirm and remediate the issue
before public disclosure.

Vernier must never require production credentials to build or test a pull
request. If a workflow appears to request them, treat that as a vulnerability.
